# 웹 접근성이란?

W3C에서는 웹 접근성을 아래와 같이 정의하고있다.

<br/>

> 웹 접근성은 웹 사이트, 도구, 기술이 장애를 가진 사용자들이 사용할 수 있도록 설계 및 개발된 것을 말합니다.
> 

추가적으로 웹 접근성을 준수하는 것은 일반 사용자 경험(UX)에도 많은 영향을 끼친다.

예를들면 밝은 야외에서 웹을 이용하여 화면이 잘 안보이는 사람, 한 손으로 음식을 먹으면서 웹을 이용하는 사용자 등 누구든 신체적 장애가 없더라도 웹을 사용하기 어려운 상황에 맞이할 수 있다.

따라서 **웹 접근성은 여러 상황에 처한 사람들이 모두 웹 서비스를 이용 가능하도록 개발하는 것**이라고 생각한다.

<br/>

# 웹 접근성을 준수해야하는 이유

1. 웹을 포함한 정보와 의사소통 기술에 대한 접근성은 유엔장애인권리협약(UN [CRPD](https://www.un.org/development/desa/disabilities/convention-on-the-rights-of-persons-with-disabilities.html))에 인간의 기본권으로 명시되어 있습니다.
2. 웹 접근성을 준수하면 다양한 사용자의 서비스 경험을 보장할 수 있다. 따라서 웹 접근성을 준수하는 것 만으로 사용자의 다양성과 만족도를 증가시킬 수 있다. 이는 브랜드와 서비스의 성장에도 긍정적인 영향을 끼칠 수 있다.
3. 웹 접근성은 법적으로 필수 요소이다.
    
    > 정보통신접근성 (Web 접근성)은 「지능정보화기본법」에 따라 장애인이나 고령자분들이 웹 사이트에서 제공하는 정보를 비장애인과 동등하게 접근하고 이용 할 수 있도록 보장하는 것으로 웹 접근성 준수는 법적의무사항 입니다. - [한국 웹 접근성 인증 평가원](http://www.wa.or.kr/m1/sub1.asp)
    >
